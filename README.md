# 📝 2022/04/14


## Linear PCM

線形PCM（パルス符号変調）データは、最も一般的な非圧縮オーディオデータ形式です。 すべての線形PCM形式の場合、mFramesPerPacketフィールドは定義上1に等しくなります。  mBytesPerPacketフィールドは、フレームあたりのバイト数に等しくなります。 すべてのパケットはバイトアラインされます。


リニアPCMオーディオの次のバリエーションは、すべてのCAFパーサーでサポートされている必要があります。


- 任意のサンプルレート
- ビッグエンディアンとリトルエンディアンの両方の16ビット、24ビット、および32ビットの符号付き整数のサンプル
- ビッグエンディアンとリトルエンディアンの両方の32ビットおよび64ビット浮動小数点のサンプル


24ビットのサンプルは、通常、PCM CAFファイル内にサンプルあたり3バイト（パック）またはサンプルあたり4バイト（アンパック）の形式で保存されます。  CAF仕様に準拠するには、両方の保存方法をサポートする必要があります。

解凍されたデータの例として、毎秒44,100フレームのサンプルレートで16ビットのビッグエンディアンステレオを記述するために、表2-2の[音声ガイド]フィールドの値を使用します。

### mFormatFlags Field

サンプルは、`2` (`00000010`)

つまり、`kCAFLinearPCMFormatFlagIsFloat`:`0` の`kCAFLinearPCMFormatFlagIsLittleEndian`:`1` ？


`signed integer` の`little`

```
enum {
    kCAFLinearPCMFormatFlagIsFloat         = (1L << 0),
    kCAFLinearPCMFormatFlagIsLittleEndian  = (1L << 1)
};
```

- kCAFLinearPCMFormatFlagIsFloat
1 for floating point, 0 for signed integer.


- kCAFLinearPCMFormatFlagIsLittleEndian
1 for little endian, 0 for big endian.
